# Журнал операций (AppLogs)

## Описание

В приложении добавлена система логирования для отладки работы в оффлайн и онлайн режимах без необходимости доступа к консоли браузера.

## Где находится

Откройте **Профиль** (иконка 👤 в нижнем меню) — журнал находится внизу страницы.

## Типы записей

| Тип | Иконка | Цвет | Описание |
|-----|--------|------|----------|
| `info` | ℹ️ | Серый | Информационные сообщения (запуск приложения, сохранение в очередь) |
| `success` | ✅ | Зелёный | Успешные операции (синхронизация, добавление данных) |
| `error` | ❌ | Красный | Ошибки (сбой синхронизации, ошибки API) |
| `warn` | ⚠️ | Оранжевый | Предупреждения (ожидание синхронизации родителей) |

## Что логируется

### При инициализации
- Запуск приложения
- Авторизация пользователя
- Загрузка данных из IndexedDB
- Статус сети (онлайн/офлайн)

### При создании объектов
- Сохранение локально (оффлайн)
- Добавление на сервер (онлайн)
- Ошибки создания

### При синхронизации
- Начало отправки очереди
- Количество операций
- Результат синхронизации
- Ошибки синхронизации

## Дополнительные функции

### Просмотр очереди синхронизации
Нажмите на строку **"Операций в очереди"** в профиле — откроется модальное окно с деталями:
- Метод операции (POST/PUT/DELETE)
- Endpoint (путь API)
- Название объекта
- Локальный ID
- Время создания

### Сброс данных (для тестирования)
В окне очереди нажмите **"🗑️ Сброс данных"** — это удалит:
- Все локальные данные (обогреватели, объекты, помещения)
- Очередь синхронизации

⚠️ **Данные на сервере НЕ будут затронуты!**

После сброса перезагрузите страницу и данные загрузятся с сервера.

### Очистка журнала
Кнопка **"🗑️ Очистить логи"** внизу журнала удалит все записи.

## Примеры использования

### Тест оффлайн работы
1. Откройте приложение
2. Отключите интернет
3. Перейдите в Профиль — увидите "Работа в оффлайн режиме"
4. Создайте объект — появится запись "Объект сохранён локально (ожидает сети)"
5. Создайте обогреватель с этим объектом — появится запись "Обогреватель сохранён (ожидает синхронизации объекта/помещения)"
6. Включите интернет
7. Нажмите "🔄 Синхронизировать"
8. Проверьте журнал — должно быть "Синхронизировано: X операций"
9. Если обогреватель был с локальным объектом, проверьте, что он синхронизировался

### Поиск проблем
- Если видите ❌ **Ошибка синхронизации** — проверьте подключение к сети
- Если операция висит в очереди — проверьте, синхронизированы ли родительские объекты
- Если ❌ **Нет токена** — войдите в систему заново
- Если видите ⚠️ **ожидает синхронизации** — дождитесь синхронизации родителей

### Сброс и повторная синхронизация
1. Профиль → нажмите "Операций в очереди"
2. Нажмите "🗑️ Сброс данных"
3. Подтвердите удаление
4. Перезагрузите страницу
5. Данные загрузятся с сервера

## Интеграция с кодом

```javascript
// Добавить лог
AppLogs.info('Сообщение');
AppLogs.success('Успех');
AppLogs.error('Ошибка');
AppLogs.warn('Предупреждение');

// Получить все логи
const logs = AppLogs.getLogs();

// Очистить логи
AppLogs.clear();

// Перерисовать журнал
renderProfile();
```

## Ограничения

- Хранится максимум 100 записей (старые удаляются)
- Журнал очищается при перезагрузке страницы (не сохраняется в IndexedDB)
- В режиме инкогнито журнал исчезнет при закрытии вкладки

## Известные проблемы

### Обогреватель не синхронизируется
Если обогреватель создан с локальным объектом/помещением, он будет ждать их синхронизации.

**Решение:**
1. Дождитесь синхронизации объекта и помещения
2. Нажмите "Синхронизировать" ещё раз
3. Проверьте журнал на наличие ошибок

### "Синхронизировано: 0 операций"
Это значит, что очередь пуста или все операции уже обработаны.

**Проверка:**
1. Нажмите "Операций в очереди"
2. Если очередь не пуста — проверьте логи на ошибки
3. Если есть ошибки "parent objects not synced" — синхронизируйте объекты сначала
